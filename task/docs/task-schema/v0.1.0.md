# task schema v0.1.0

> **状态**：unstable（v0.x.x 自由演化期），breaking changes expected
>
> **生效日期**：2026-05-04

## 字段总览（8 个）

```yaml
---
aliases:
  - 买牛奶                          # 复用 Obsidian aliases，承担 title
tags:
  - task
  - task/status/inbox
  - task/context/errand             # 仅当 task_context 有值时
task_status: inbox                  # core，必填
task_created: 2026-05-04T12:26      # core，必填，ISO 8601 含时分
task_updated: 2026-05-04T12:26      # core，必填
task_context: errand                # 选填
task_due: 2026-05-05                # 选填
task_source: chat                   # 选填，自由文本，可写 [[wikilink]]
task_schema: 0.1.0                  # core，必填
---
```

## 字段定义

### `aliases`（Obsidian 原生）

- 类型：列表
- 必填：是（至少 1 项）
- 用途：承载 task 的完整 title（一般是中文）
- 示例：`["买牛奶"]` / `["写完 Q2 总结报告"]`

理由：Obsidian 已经原生识别 aliases，可以在 Quick Switcher、双链补全里搜到；自造 `task_title` 字段反而失去 Obsidian 红利。

### `tags`（Obsidian 原生）

- 类型：列表
- 必填：是
- 必含：`task`
- 必含：`task/status/<当前 task_status>` —— 与 `task_status` 字段值同步
- 选含：`task/context/<当前 task_context>` —— 仅当 `task_context` 有值时同步
- 不要再加其他 `task/*` 标签（避免和 frontmatter 双源不一致）

### `task_status`（core）

- 类型：枚举字符串
- 取值：`inbox` | `todo` | `doing` | `blocked` | `done` | `dropped`
- 必填：是
- 修改时机：仅由 `task-operate` / `task-organize` 修改
- 修改规则：必须遵循下方状态机

### `task_created`（core）

- 类型：ISO 8601 字符串，**含时分**
- 格式：`YYYY-MM-DDTHH:mm`（如 `2026-05-04T12:26`）
- 必填：是
- 修改时机：**仅在 collect 时一次性写入**，之后任何 skill 都不许改

### `task_updated`（core）

- 类型：ISO 8601 字符串，含时分
- 格式：`YYYY-MM-DDTHH:mm`
- 必填：是
- 修改时机：**任何修改 frontmatter 的操作必须同步更新此字段**

### `task_context`（选填）

- 类型：自由文本
- 推荐值：`@home` / `@computer` / `@errand` / `@phone` / `@office` 或自己约定的项目名
- 修改时同步更新 `tags` 中的 `task/context/<value>`

### `task_due`（选填）

- 类型：日期字符串 `YYYY-MM-DD`（不要时分）
- 用途：deadline；review 时用来算 `overdue_count`

### `task_source`（选填）

- 类型：自由文本
- 推荐值：
  - 来源类型：`chat` / `email` / `meeting` / `idea` / `manual`
  - Obsidian wikilink：`[[ideas/task-management-skill-suite/brainstorm]]`
  - 路径：`ideas/task-management-skill-suite/brainstorm.md`
- 不做结构化双向链；写成 wikilink 由 Obsidian 自然反向链接

### `task_schema`（core）

- 类型：版本号字符串
- 当前值：`0.1.0`
- 写入时机：仅在 collect 时一次性写入
- **任何 skill 在常规操作中不得修改此字段**（专属 schema migration 工具除外）

## 状态机

```
            ┌──────────────┐
            │    inbox     │
            └───┬──────┬───┘
        organize│      │drop
                ▼      ▼
            ┌──────┐ ┌─────────┐
        ┌──→│ todo │ │ dropped │  ← 终态
        │   └──┬───┘ └─────────┘
   重开 │      │ start / block / done / drop
        │      ▼
        │  ┌───────┐  block   ┌─────────┐
        │  │ doing │ ───────→ │ blocked │
        │  └───┬───┘ ←─────── └────┬────┘
        │      │  unblock          │
        │      │ done / drop       │ unblock / drop
        │      ▼                   ▼
        │  ┌──────┐
        └──┤ done │  ← 终态（重开则回 todo）
           └──────┘
```

合法转换边（写死，task-operate 校验时严格按此表）：

| 来源 | 允许的目的 |
|---|---|
| `inbox` | `todo` / `dropped` |
| `todo` | `doing` / `blocked` / `dropped` / `done` |
| `doing` | `blocked` / `done` / `todo` |
| `blocked` | `todo` / `doing` / `dropped` |
| `done` | `todo`（视为重开） |
| `dropped` | `todo`（视为重开） |

非法示例：

- `done → blocked` ❌（要先回 `todo` 再 block）
- `inbox → done` ❌（必须先 organize 进入 active）
- `inbox → doing` ❌（同上）

## 文件名规则

格式：`<YYYYMMDD-HHMM>-<slug>.md`

- 时间戳：12 字符 `YYYYMMDD-HHMM`，与 `task_created` 同时刻
- 分隔符：单个 `-`
- slug：kebab-case，2–5 个英文词
  - 中文 task：取关键词翻译为简短英文 slug（"买牛奶" → `buy-milk`）
  - 同分钟重名：在 slug 末尾加 `-2` / `-3`（详见 task-collect SKILL.md）

## 物理目录（三段对称）

```text
tasks/
├── inbox/                    # collect 落点（task_status: inbox）
│   └── 20260504-1226-buy-milk.md
├── active/                   # 工作区（task_status: todo / doing / blocked）
│   └── 20260504-0900-write-report.md
└── archived/                 # 归档（task_status: done / dropped）
    └── 2026-05/
        └── 20260503-2200-call-mom.md
```

约定：

- `task_status` 与物理目录必须一致（task-operate 前置检查 #7 强制）
- `inbox → active` 由 task-organize 搬
- `active → archived/YYYY-MM/` 由 task-operate 在 done / dropped 时搬
- `archived` 下按月分子目录，避免单目录文件爆炸

## ID 提取规则

- **不存** `task_id` 字段
- ID = 文件名前 13 字符（`YYYYMMDD-HHMM`）
- 跨系统引用 task **一律用完整相对路径**（如 `tasks/active/20260504-1226-buy-milk.md`），不只用 ID
- 这样做的代价：task 文件改名时引用会断；好处：完全免去 ID 维护

## 容忍机制（v0.x.x 强制要求）

所有 task-* skill 在读取 task 文件时**必须**：

1. 容忍未知字段（不报错，不删除）
2. 容忍缺失的选填字段
3. 对缺失的 core 字段（`task_status` / `task_created` / `task_updated` / `task_schema`）：
   - 读取时给出警告，但仍尝试继续
   - 写回时按当前版本补齐
4. 容忍 `task_schema` 值低于当前版本（视为旧文件，按当前 skill 行为处理；不主动 migrate）
